# Zettelkasten知識ベース

このディレクトリは、Zettelkasten方式で知識を管理します。

## Zettelkastenとは

Zettelkastenは、ドイツの社会学者ニクラス・ルーマンが開発した知識管理システムです。
小さなノート（Zettel）を相互にリンクさせることで、知識のネットワークを構築します。

## ノートの命名規則

各ノートは、タイムスタンプベースのIDを使用します：

```
YYYYMMDDHHMMSS_short_description.md
```

例：`20240101000000_kaggle_basics.md`

## ノートの構造

```markdown
---
id: 20240101000000
title: Kaggle基本戦略
tags: [kaggle, strategy, basics]
links: [20240102000000, 20240103000000]
created: 2024-01-01
updated: 2024-01-15
---

# Kaggle基本戦略

## 内容
（知識の内容）

## 関連ノート
- [[20240102000000]] 特徴量エンジニアリング
- [[20240103000000]] モデル選択

## 関連タスク
- [ ] タスク1
- [ ] タスク2

```

## リンクの作成

Obsidianでは、`[[ノートID]]` の形式でリンクを作成できます。
これにより、知識間の関係性を可視化できます。

## タスクへの変換

知識ノートからタスクを生成する際は、以下の手順を踏みます：

1. 知識ノートを読み、実装可能なタスクを特定
2. `knowledge/tasks/inbox/` に新しいタスクを作成
3. タスクのメタデータに `related_notes` として元のノートIDを記録
4. タスクをJSON形式に変換してマルチエージェントシステムに引き渡し

## フィードバックループ

1. **知識収集**: 新しい知識をノートとして記録
2. **タスク生成**: 知識から実装可能なタスクを生成
3. **実験実行**: マルチエージェントシステムで実験を実行
4. **学びの記録**: 実験結果を知識ノートに反映
5. **新しい知識**: 学びから新しい知識ノートを作成

この循環により、知識とタスクが有機的に成長していきます。
