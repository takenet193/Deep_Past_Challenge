# タスク: MCP環境の設定と運用フロー整理

## 目的

このプロジェクトで利用するMCP（Model Context Protocol）の設定を整備し、  
どのMCPサーバーをどの用途で使うかを明確にし、再現性のあるセットアップ手順を残す。

## 完了条件

- プロジェクト用のMCP設定ファイルが作成され、Gitで管理されている
- 利用するMCPサーバー（例: ローカルツール、外部API連携 など）が一覧化されている
- 新しくMCPサーバーを追加／更新／削除する手順がドキュメント化されている

## 手順（チェックリスト）

1. 現在使用している / 使用予定のMCPサーバーを洗い出す
   - [ ] ローカル開発用MCP（ファイル操作・検索など）
   - [ ] 外部サービス連携用MCP（必要であれば）
2. プロジェクト用のMCP設定ファイルを整備する
   - [ ] `.cursor/` 配下の設定ファイル構成を確認する
   - [ ] MCPサーバーのエンドポイント・プロトコル・認証情報の管理方法を決める
   - [ ] 必要な設定ファイルを作成し、Gitで管理可能な形にする
3. 運用フローのドキュメント化
   - [ ] MCPサーバーを新規追加する手順をメモにする
   - [ ] MCP設定を更新する際の注意点（APIキー管理、環境ごとの差分など）を整理する
   - [ ] 不要になったMCPサーバーを無効化／削除する手順を記載する
4. 動作確認
   - [ ] Cursor からMCPが正しく呼び出せるかを確認する
   - [ ] 代表的なユースケース（例: ファイル操作、検索、タスク管理連携など）が期待通り動くかを確認する

## メモ

- セキュリティ上、秘匿情報（APIキー等）は `.env` やOSのキーチェーンなどで管理し、リポジトリには含めない
- 将来的に他のKaggleプロジェクトでも再利用できるよう、できるだけ汎用的な構成にする
