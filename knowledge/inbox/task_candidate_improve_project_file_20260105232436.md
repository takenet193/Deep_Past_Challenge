# （タスク候補）プロジェクトファイルを改良する（今回の実験の教訓を反映）

## 目的
- 今回の実験で分かった「運用上の痛み/改善点」を、プロジェクトノート（projectファイル）に反映して、次回以降の迷い・手戻りを減らす

## 完了条件
- プロジェクトノートに以下が追記/整理されている
  - 今回の実験の「教訓（Lessons Learned）」セクション
  - 次の実験のための「改善アクション（Next Actions）」が箇条書きで明確
  - 重要リンク（関連タスク/重要ノート/データ場所/評価指標）が更新されている

## 手順（ラフでOK）
- [ ] 今回の実験で起きた問題/判断/気づきを箇条書きで収集（5〜10個）
- [ ] それを「再発防止のルール」「テンプレ改善」「手順改善」に分類
- [ ] projectノートに反映（追記 + 既存セクションの整形）
- [ ] 関連タスク/ノートへのリンクを追加（必要なら相互リンク）

## メモ
- 確定したら `knowledge/tasks/{active|waiting|someday}/` にコピーしてSSOT化し、元ファイルは `knowledge/inbox/archive/` へ
