README.md, docs 配下に大量の .md ファイルがありますが、重複した記述や冗長な記述が多いため、不要な部分を削除・整理してください。
以下を守る形で整理してください。最終確認も行ってほしいです。

- README.md はプロジェクトの概要、セットアップ手順、docs 配下の .md ファイルへのリンクに集中させてください。
- docs 配下の .md ファイル内で行われているロードマップは必要であれば docs/roadmap.md に転記・整理してください。それ以外の .md ファイルからはロードマップ・タスク・進捗に関する記述を削除してください。
- ある .md ファイル内で述べられている内容は、他の .md ファイルで繰り返し説明しないでください。とにかく構造化と簡潔さを重視してください。
- 読み手が必要な記述を最低限のアクセスで理解できるように、README.md と各 .md を簡潔に保ってください。



docs 配下の .md ファイル（サブディレクトリ含む）をすべて確認し、シンプルな記述に再構成してください。

- 中身が実質ないと言えるセクションは削除してください。
- 実質的に重複している・別のセクションと簡単に統合できるセクションは削除・統合してください。
- 本質的に docs 内で記述すべき例は残して構いませんが、実際のコードへの参照で済む場合はコードへの参照に置き換えて、冗長な説明は避けてください。
